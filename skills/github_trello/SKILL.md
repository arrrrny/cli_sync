# GitHub-Trello Integration Skill

## Overview
This skill enables seamless integration between GitHub repositories and Trello boards, allowing for effective project management and issue tracking across both platforms.

## Capabilities

### 1. Issue Linking
- Link GitHub issues to Trello cards
- Link GitHub pull requests to Trello cards
- Create bidirectional connections between platforms

### 2. Attachment Management
- Attach GitHub URLs as formal links to Trello cards
- Connect both issues and pull requests to relevant cards
- Maintain organized references between platforms

### 3. Cross-Platform Synchronization
- Establish clear connections between GitHub issues and Trello cards
- Maintain traceability for project tracking
- Enable collaborative workflows across platforms

## Usage Patterns

### Linking GitHub Issues to Trello Cards
1. Identify the GitHub issue that corresponds to a Trello card
2. Add a comment to the Trello card with the GitHub issue URL
3. Optionally attach the GitHub URL as a formal link attachment

### Managing Pull Requests
1. Connect GitHub pull requests to relevant Trello cards
2. Use attachments for formal linking
3. Maintain clear references for code review workflows

### Best Practices
- Use consistent naming conventions for linked items
- Maintain clear descriptions in both platforms
- Regularly update linked items to reflect current status
- Use attachments for formal connections and comments for additional context

## Benefits
- Enhanced project visibility across platforms
- Improved collaboration between team members
- Better traceability from requirements to implementation
- Streamlined workflow management
- Reduced risk of disconnected tasks

## Implementation Notes
- The integration works by creating explicit links between GitHub and Trello
- Both comment-based and attachment-based linking methods are supported
- No special Power-Ups required for basic functionality
- Works with both GitHub issues and pull requests
